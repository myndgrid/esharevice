# Feature: A11y deep pass

**Created:** 2026-05-16 16:57 UTC
**Last Updated:** 2026-05-16 16:57 UTC
**Status:** Live. Five targeted improvements landed after a full audit: skip-to-content link in the root layout, aria-label + label on the conversation composer, role/aria-live on the message log so SR announces new SSE messages, aria-busy on the Send button instead of replacing its text with "…", and converting the home + saved card grids from `<div>`-of-`<Link>`s to semantic `<ul>`/`<li>` structure. Lighthouse a11y stays at 100; the wins here are real-screen-reader-experience improvements that don't move automated scores.

## Overview

The codebase came in healthier than the original backlog item assumed:

- Lighthouse a11y was already 100/100
- Every `<Button>` had visible text content (no icon-only buttons)
- Icon-only links (save bookmark, theme toggle, header profile avatar, mobile tab bar) all had aria-labels
- Form fields used the wrapped-`<label>` pattern correctly
- Every page mounts a `<main>` landmark
- No modals exist yet — focus traps are N/A for this iteration

The real audit gaps were five places where automated tooling can't tell something is wrong because the markup is syntactically valid but semantically incomplete. Each fix targets one specific screen-reader experience.

## Fixes

| File | What changed |
|---|---|
| [apps/web/app/layout.tsx](../../apps/web/app/layout.tsx) | Skip-to-content `<a>` is the first focusable element on every page. Hidden with `sr-only` until it receives keyboard focus, then `focus:not-sr-only` reveals it as a high-contrast pill. Target is a wrapper `<div id="main-content" tabIndex={-1}>` around `{children}` — `tabIndex={-1}` makes it programmatically focusable without entering the tab order, so activating the skip link moves focus past Header + tab bar straight into page content. |
| [apps/web/app/messages/[id]/conversation-view.tsx](../../apps/web/app/messages/[id]/conversation-view.tsx) | Composer `<textarea>` gains a visually-hidden `<label htmlFor>` ("Message"). Previous state was placeholder-only — SR users got no field name on focus. Send button now keeps the text "Send" through both idle + pending states; `aria-busy={sending}` signals in-progress instead of swapping the text for "…" (which SR reads as "horizontal ellipsis" or nothing, depending on the engine). |
| [apps/web/app/messages/[id]/conversation-view.tsx](../../apps/web/app/messages/[id]/conversation-view.tsx) | Message `<ul>` gains `role="log"` + `aria-live="polite"` + `aria-label="Conversation messages"`. New messages arriving via the SSE stream now get announced by screen readers. `polite` (not `assertive`) waits for any in-progress speech to finish — prevents interrupting the user mid-read of an earlier message. |
| [apps/web/app/page.tsx](../../apps/web/app/page.tsx) | Home grid: `<div className="grid">` → `<ul aria-label="Exchange listings">` with each card wrapped in `<li>`. SR users hear "list, N items" instead of "twelve links" without context. |
| [apps/web/app/saved/page.tsx](../../apps/web/app/saved/page.tsx) | Same conversion for the saved-items grid. |

## What stayed unchanged

- All other `<Button>` uses: text content is sufficient; no change needed.
- Mobile tab bar nav: already has `aria-label="Primary"` + sr-only unread-count text.
- Header profile avatar link: already `aria-label="Open profile"`.
- ThemeToggle: already has stateful `aria-label`.
- SaveButton: already `aria-pressed` + state-aware `aria-label`.
- Form Field component: wrapped-`<label>` pattern means the input is associated with its label by DOM containment; the redundant `htmlFor` is harmless.

## Edge Cases & Gotchas

- **`role="log"` is the right ARIA role for a chat-style append-only stream.** `role="feed"` would imply we paginate by scroll position (which we kind of do, but `log` is more idiomatic for chronological message lists).
- **`aria-live="polite"` queues announcements** — if 3 messages arrive in rapid succession, SR reads them in order. `assertive` would interrupt; for a chat-like flow that's overkill.
- **Skip link uses `href="#main-content"`, not `focus()`.** Browsers handle the focus jump natively when the target has `tabIndex={-1}`. Doing it via JS would not work for users with JS disabled.
- **Skip link is the first element in `<body>`, BEFORE `<Header>`.** It's the first thing Tab reaches, which is the whole point — pressing Tab once on page load lands here, then Tab again skips it. Activating skips past Header.
- **Skip-link target wraps `{children}` in the layout, not the page-level `<main>`.** Avoids touching every page; preserves the `<main>` landmark inside the wrapper. The `tabIndex={-1}` keeps the wrapper out of tab order.
- **`aria-busy={sending}` only applies on the button.** I deliberately didn't put it on the form-wrapper or the input, because a busy form would suggest the input is also unusable — the textarea is still interactive while a previous send is in flight.
- **Card-grid `<ul>` uses `aria-label`, not a visible label.** The grid is semantically a list but visually a card grid; the label tells SR what the list contains without crowding the visual layout.
